Introduction back to ToC

In the era of digitalization, data ecosystems are evolving to enable data sharing across networks of distributed organizations. Against this backdrop, digital identity management requires a standardized yet flexible approach in cross-organizational environments.

However, digital identity management remains fragmented. This is because identities are diverse, and ensuring identity interoperability is challenging. The heterogeneity creates silos that hinder secure cross-organizational interaction. For instance, a system using Security Assertion Markup Language (SAML) may not natively process a JSON Web Token (JWT) as an assertion. Additionally, some specific protocols lack the capability of issuing verifiable assertions, such as Lightweight Directory access Protocol (LDAP).

To address the problem of identity interoperability within data ecosystems, we propose DIDL, a description language to syntactically and semantically represent diverse identity concepts, ensuring technical and semantic interoperability across different data ecosystems.

Figure 1 depicts the core information model underlying the description language. DIDL is structured around the three core elements: identifier (didl:Identifier), credential (didl:Credential), and assertion (didl:Assertion), each enriched with metadata that describes an identity and supports extensibility. The basic class, didl:Identity, is associated with key attributes such as issuer (didl:issuer), issuance data (didl:issueDate), validity period (didl:validFrom and didl:validTo), and status (didl:status). Together, the basic class references a subject (didl:IdentitySubject) it identifies.

Identifier back to ToC

Identifiers are the foundational elements of digital identity, providing unique references to identity subjects. DIDL supports describing both simple identifiers (didl:Identifier) or identifier collection (didl:IdentifierCollection). The former is associated with didl:identifierFormat (representing the basic format standard) and didl:functional (representing its specific use). The latter combines different identities for application-specific identity usage.

JSON-LD Example

{
  "@context": "https://w3id.org/didl/context.jsonld", 
  "@type": "Identifier",
  "uid": "urn:order:123",
  "identifierFormat": "URN",
  "namespace": {
    "@type": "xsd:string",
    "@value": "urn:order"
  },
  "identifierFunctional": "OrderID"
}

Credential back to ToC

Credentials (didl:Credential) represent evidence of a subjectâ€™s identity, indicating authentication methods using various formats (didl:CredentialFormat). DIDL specifies credentials under three types: knowledge, possession, and inherence. Moreover, credentials are associated with attributes (didl:Attribute) to present additional identity-related information in key value pairs, using didl:credentialAttribute.

JSON-LD Example


{
  "@context": "https://w3id.org/didl/context.jsonld",
  "@type": "Credential",
  "credentialFormat": "PasswordCredential",
  "credentialAttribute": [
    {
        "key": "Username",
        "value": {"@type": "xsd:string", "@value": "user123"}
    },
    {
        "key": "Password",
        "value": {"@type": "xsd:string", "@value": "pass123"}
    }
  ]
}

Assertion back to ToC

Assertions (didl:Assertion), resulted from an authentication process, are statements that convey verifiable identity attributes used for authorization. These attributes are associated using didl:assertedAttribute. Specifically, assertions can be associated with cryptographic methods using sec:Proof, which ensures the validity and integrity of assertions. Thus, they can also be used for authentication. DIDL supports both simple assertions and composite assertions (didl:AssertionCollection).

JSON-LD Example


{
    "@context": [
        "https://w3id.org/didl/context.jsonld", 
        https://w3id.org/security/suites/ed25519-2020/v1"
    ],

    "uid": "27ea6480-b0c3-446e-88a6-ff3ac7e8cac0",

    "validTo": "2026-01-12T00:00:00",
    "status": "Active",
    "assertionFormat": "JsonWebToken",

        {
            "key": "FirstName",
